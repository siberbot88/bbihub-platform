üìã Best Practices

‚ö†Ô∏è ALWAYS specify which repository

When working with Git commands, always navigate to the specific folder (backend or mobile) first.

üí° Commit Separately

Changes in backend and mobile should be committed to their respective repositories. Don't assume changes in one repo affect the other.

Recommended Workflow:

  1. Make changes in backend or mobile
  2. Navigate to the specific folder (cd backend or cd mobile)
  3. Check status: git status
  4. Review changes: git diff
  5. Stage changes: git add .
  6. Commit: git commit -m "Descriptive message"
  7. Push: git push origin main

üîß Troubleshooting

Git Ownership Warning

If you see a "dubious ownership" error, run:

git config --global --add safe.directory E:/BBIHUB/backend
git config --global --add safe.directory E:/BBIHUB/mobile

Check Remote URLs

To verify which GitHub repository each folder is connected to:

# For backend
cd E:\BBIHUB\backend
git remote -v

# For mobile
cd E:\BBIHUB\mobile
git remote -v
